.NET Core Interview Questions and Answers

  • 1. What is .NET Core?

    .NET Core is a cross-platform, open-source framework for building modern, cloud-based, and internet-connected applications. It is a successor to the .NET Framework and is designed to work on Windows, macOS, and Linux.

  • 2. What are the key differences between .NET Framework and .NET Core?

    Key differences include cross-platform support (Linux, macOS, and Windows) in .NET Core, modularity and lightweight nature, and open-source status. .NET Framework is Windows-only and has a more monolithic structure.

  • 3. What is the purpose of the Startup class in .NET Core?

    The Startup class is used to configure services and the application pipeline. It contains methods like ConfigureServices and Configure, which define how the application should respond to HTTP requests and configure services used by the application.

  • 4. What is Dependency Injection in .NET Core?

    Dependency Injection (DI) is a design pattern used in .NET Core to achieve Inversion of Control (IoC). It allows the injection of dependencies (such as services and repositories) into classes rather than hard-coding them, improving modularity and testability.

  • 5. What is Middleware in .NET Core?

    Middleware is a component in .NET Core that is used to handle HTTP requests and responses. Middleware components are assembled into an application pipeline and can perform tasks such as authentication, logging, and exception handling.

  • 6. What is Entity Framework Core?

    Entity Framework Core (EF Core) is an Object-Relational Mapper (ORM) for .NET Core that provides a way to interact with databases using .NET objects. It supports LINQ queries, change tracking, and schema migrations.

  • 7. What is a Razor Page?

    A Razor Page is a page-centric model in ASP.NET Core that simplifies the development of page-based web applications. It combines the model and view into a single file with a `.cshtml` extension and is part of the Razor Pages framework.

  • 8. How does .NET Core handle configuration?

    .NET Core uses a configuration system that supports various sources, such as JSON files, environment variables, and command-line arguments. Configuration settings are typically stored in `appsettings.json` and accessed using the `IConfiguration` interface.

  • 9. What is Kestrel?

    Kestrel is a cross-platform web server for ASP.NET Core applications. It is designed to be fast and lightweight and is used as the default web server for .NET Core applications. It can be used in combination with a reverse proxy server like IIS or Nginx.

  • 10. What is the role of `Program.cs` in .NET Core?

    The `Program.cs` file contains the entry point for a .NET Core application. It is responsible for setting up the host, which includes configuring the web server and loading configuration settings before the application starts.

  • 11. What is the difference between `.NET Core` and `.NET Standard`?

    .NET Core is a runtime and framework for building cross-platform applications, while .NET Standard is a specification that defines a set of APIs that all .NET implementations should support. .NET Standard allows for code sharing across different .NET platforms.

  • 12. What are the common file types used in .NET Core applications?

    Common file types include `.csproj` (project file), `.cshtml` (Razor pages), `.json` (configuration files), and `.dll` (compiled assemblies). The `.csproj` file is used to define project properties and dependencies.

  • 13. How does .NET Core support logging?

    .NET Core supports logging through a built-in logging framework that can log messages to various outputs such as console, files, and external systems. The `ILogger` interface is used to write log messages, and logging providers are configured in the `Startup` class.

  • 14. What is ASP.NET Core Identity?

    ASP.NET Core Identity is a membership system that provides user authentication and authorization functionalities. It includes features for managing user accounts, roles, and claims, and integrates with the ASP.NET Core security framework.

  • 15. What is the purpose of `appsettings.json`?

    `appsettings.json` is a configuration file used to store application settings in a structured format. It allows you to define key-value pairs that can be accessed throughout the application using the `IConfiguration` interface.

  • 16. What is a Scoped service in .NET Core?

    A Scoped service is a type of service that is created once per request (or per scope) in a web application. It is used to maintain state within a single request and is often used for database contexts and other request-specific services.

  • 17. What is `ConfigureServices` in .NET Core?

    The `ConfigureServices` method in the `Startup` class is used to register services and configure dependency injection for the application. It is called at the start of the application and is where you configure services such as MVC, Entity Framework, and custom services.

  • 18. What is the difference between `IApplicationBuilder` and `IServiceCollection`?

    `IApplicationBuilder` is used to configure the application's request pipeline, such as adding middleware components. `IServiceCollection` is used to register services for dependency injection and configure the application's services.
